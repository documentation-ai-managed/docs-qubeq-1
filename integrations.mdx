## Git Integration

Connect Qubeq to your Git provider to automatically import and sync documentation from repositories.

<Steps>
  <Step title="Connect Repository" icon="git-branch">
    Navigate to Settings > Integrations in your Qubeq dashboard.
    
    Select GitHub, GitLab, or Bitbucket and authorize the connection.
  </Step>
  <Step title="Select Repository" icon="folder">
    Choose the repository containing your Markdown files.
    
    Specify the branch (e.g., `main`) and path (e.g., `docs/`).
  </Step>
  <Step title="Enable Sync" icon="refresh-cw">
    Toggle auto-sync and set the interval (e.g., every 5 minutes).
    
    Test the connection to pull initial content.
  </Step>
</Steps>

<Callout kind="tip">
  Use `.qubeqignore` files in your repo to exclude sensitive files from syncing.
</Callout>

## Webhook Setups

Set up webhooks to notify external services of changes in Qubeq, such as page publishes or updates.

### Configuration Steps

<Tabs>
  <Tab title="GitHub Actions" icon="github">
    Add this to your workflow YAML:
    
````yaml
name: Qubeq Sync
on:
  webhook:
    url: https://api.qubeq.com/webhooks/your-webhook-id
    events: [page.updated, page.published]
jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Update Docs
        run: |
          # Your sync logic here
````
  </Tab>
  <Tab title="Zapier" icon="zap">
    In Zapier, create a new Zap with Qubeq as the trigger app.
    
    Select event (e.g., "New Page Update") and paste your webhook URL:
    
    ```
    https://api.qubeq.com/webhooks/your-webhook-id
    ```
    
    Configure actions like Slack notifications or Google Sheets updates.
  </Tab>
</Tabs>

## API Access

Build custom integrations using Qubeq's REST API. All endpoints require authentication via Bearer token.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer `{YOUR_API_TOKEN}`
</ParamField>

<ParamField query="spaceId" param-type="string" required="true">
  Your documentation space identifier.
</ParamField>

### Example: List Pages

<Request tabs="JavaScript,cURL" show-lines="true">
  ```javascript
  const response = await fetch('https://api.qubeq.com/v1/spaces/{spaceId}/pages', {
    headers: {
      'Authorization': 'Bearer YOUR_API_TOKEN'
    }
  });
  const pages = await response.json();
  console.log(pages);
  ```
  ```bash
  curl -H "Authorization: Bearer YOUR_API_TOKEN" \
       "https://api.qubeq.com/v1/spaces/{spaceId}/pages"
  ```
</Request>

<Response tabs="200">
  ```json
  {
    "pages": [
      {
        "id": "page-123",
        "title": "Introduction",
        "path": "/introduction",
        "updatedAt": "2024-10-15T10:00:00Z"
      }
    ]
  }
  ```
</Response>

## Embedding External Content

Embed dynamic content from tools like Figma, Notion, or custom iframes.

<Expandable title="Advanced Embedding Options" default-open="false">
  Use the Embed block in the editor:
  
  1. Paste the external URL (e.g., Figma prototype).
  2. Qubeq fetches and renders interactive content.
  3. Set refresh intervals for live updates.
  
  Supported providers:
  
  | Provider | Example URL |
  |----------|-------------|
  | Figma    | https://www.figma.com/file/... |
  | Notion   | https://notion.so/embed/...   |
  | Google Docs | https://docs.google.com/... |
  
  <Callout kind="alert">
    Embeds respect CORS policies. Test public accessibility first.
  </Callout>
</Expandable>
